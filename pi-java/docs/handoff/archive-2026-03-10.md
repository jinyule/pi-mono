This file is a normalized archive summary for the period before the handoff docs were split by phase. The original long-form markdown became unreadable after an encoding issue, so this file preserves the important continuation state in a compact form.

## Snapshot as of 2026-03-10

- Phase 0: complete.
- Phase 1 (`pi-ai`): complete.
- Phase 2 (`pi-agent-runtime`): complete.
- Phase 3 (`pi-session`): complete.
- Phase 4 (`pi-tools`): complete.
- Phase 5 (`pi-extension-spi`): complete.
- Phase 6 (`pi-tui`): brought to usable baseline and then completed the same day.
- Phase 7 (`pi-cli` / `pi-sdk`): initial functional bring-up had started by the end of 2026-03-10.

## What was already done by then

### `pi-ai`

- Core model and event types.
- `EventStream`, `AssistantMessageEventStream`, registries, credential resolution, `PiAiClient`.
- Provider transport primitives and the first provider implementations.
- Provider behavior tests.

### `pi-agent-runtime`

- Core loop skeleton.
- Sequential tool execution.
- Steering / follow-up queues.
- Stateful `Agent` facade.
- Abort and lifecycle handling.

### `pi-session`

- Session model and JSONL codec.
- Session migrations and replay.
- `SessionManager`, branching, fork extraction, persisted append behavior.
- Settings layering and instruction resource loading.

### `pi-tools`

- Shared primitives.
- `read`, `write`, `edit`, `bash`, `grep`, `find`, `ls`.
- Golden and contract tests for tool output shape.

### `pi-extension-spi`

- Core extension contracts.
- Discovery and isolated loading.
- Event bus and resource discovery.
- Reload runtime and example plugin.

### `pi-tui`

- Core contracts and process terminal.
- Diff renderer, overlay stack, text primitives.
- `Input`, `Editor`, `Markdown`, `Loader`, `SelectList`, `SettingsList`, `Image`, `VirtualTerminal`.

### `pi-cli` / `pi-sdk` (initial state by 2026-03-10)

- CLI argument parser and startup dispatcher.
- `interactive`, `print`, `json`, and `rpc` modes.
- SDK facade and session bootstrap.
- `list-models`, session resolution, `--resume`, and `--export`.
